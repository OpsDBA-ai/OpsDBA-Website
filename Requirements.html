Technical Prerequisites

OpsDBA is designed to be lightweight and non-intrusive. Review the environment requirements below to ensure a smooth deployment.

Core Infrastructure

Required for both Master and Client deployments.

SQL Server 2016+

Must be SQL Server 2016 (SP2) or newer. We rely on modern DMVs and T-SQL functions for diagnostics.

Standard Enterprise Developer

sysadmin Access

Required only during initial installation to create the OpsDBA database, Login, and Agent Jobs.

Database Mail

Critical Requirement. Must be enabled on Master AND Client servers. Used for the Daily DBA Summary Report (Master) and Internal SQL Alerts (Clients).

Mixed Mode Auth

SQL Server & Windows Authentication mode must be enabled. We use a standard SQL Login to bridge connections across untrusted domains.

Optional Feature

AI Insights Engine

By default, the AI Bridge is disabled. To unlock automated root-cause analysis for your error logs, you must configure a supported provider.

PowerShell Requirement

If (and only if) AI features are enabled, the Master Server requires:

  • Outbound Internet Access (HTTPS/443) to the AI Provider API.
  • Permission for SQL Agent to run PowerShell job steps.

Technical Deep Dive: Model Selection

While OpenAI and Microsoft Copilot remain the "Industry Standards" for enterprise, Google's aggressive Q4 2025 updates have introduced a highly efficient alternative that is uniquely suited for database log analysis.

1. Google Gemini 3.0 Flash Released Q4 2025

  • The "Killer Feature": 1 Million Token Window

    SQL Server Error Logs are massive. Other models often have to "truncate" (cut off) the log to fit it into memory. Gemini can ingest the entire log history in a single pass without forgetting context.

  • Efficiency Revolution

    The new "Flash" architecture (updated Dec 2025) is designed explicitly for high-volume, low-latency tasks. It delivers reasoning capabilities comparable to larger models but at a fraction of the compute cost.

  • Why it's Free

    Because this model is so computationally efficient, Google offers a generous Free Tier that easily covers the volume of a typical OpsDBA deployment (checking logs every 4 hours), making this a zero-cost add-on.

2. OpenAI (GPT-4o)

  • The Gold Standard

    GPT-4o remains the benchmark for reliability and reasoning speed. It is the "safe bet" for organizations that have already standardized their procurement on OpenAI's API.

  • Paid vs. Free

    Unlike Google, OpenAI focuses on paid Enterprise tiers. You will need to attach a credit card to your API account, though the cost for OpsDBA's text-only payload is typically very low (< $5/month).

3. Microsoft Copilot (Azure)

  • Private Endpoints

    We connect securely to your internal Azure OpenAI endpoints. Traffic does not traverse the public internet.

  • Compliance

    Architecturally distinct from public OpenAI. Your data is handled according to your existing Azure Enterprise Agreement (EA) and compliance standards (HIPAA, SOC2, etc.).

4. Ollama (Local LLM)

  • Air-Gapped Security

    We run open-source models (like Llama 3) locally on your Master Server. No data ever leaves your network. This is the only option for environments with zero internet access.

  • Hardware Requirements

    Requires significant local compute. Your Master Server will need 16GB+ RAM dedicated to the model, and ideally a GPU for acceptable inference speed.
