Technical Prerequisites

OpsDBA is designed to be agentless and non-intrusive. We leverage native SQL Server features to minimize impact on your production environment.

SQL

SQL Server 2016 - 2022

Compatible with Standard and Enterprise editions. We support both Default and Named Instances with dynamic performance counter resolution.

Standard Enterprise Named Instances

Database Mail

Database Mail is required for real-time alerts. We utilize the msdb alerting engine to trigger the OpsDBA Responder for zero-latency notification.

Note: Required for InstantAlerts delivery.

No OS Agents

OpsDBA is 100% T-SQL based. We require zero Windows-level agents, third-party executables, or service reboots for installation.

Mixed Mode Auth

Required for cross-domain Master-to-Client communication. We use a secure, low-privilege SQL Login with strictly defined read-only permissions.

Security by Redaction

We sanitize your technical data locally before it ever leaves your Master Server.

Edge-Side Redaction

Before sending a log for AI-enhanced diagnosis, our logic scrubs sensitive metadata:

  • IP Addresses (Public & Private)
  • File Paths & User Accounts
  • Server & Database Names

Supported AI Models

AI features are disabled by default. If enabled, the Master Server requires outbound HTTPS access to your provider API endpoint.

Google Gemini

Our primary recommendation. Supports Gemini 1.5 Pro with a massive token window for deep log history analysis.

OpenAI

Standard high-precision troubleshooting. Supports the latest GPT-4o models via standard API.

Microsoft Copilot

Enterprise compliance. Access LLMs through your private Azure OpenAI Service tenant.

Ollama

The air-gapped solution. Run local models like Llama 3 directly on your server hardware.
