Technical Prerequisites

OpsDBA is designed to be agentless and non-intrusive. We leverage native SQL Server features to minimize impact on your production environment.

SQL

SQL Server 2016 - 2022

Compatible with SQL Server 2016 (SP2) and newer. We utilize modern Dynamic Management Views (DMVs) for non-blocking performance diagnostics.

Standard Enterprise

Database Mail

Must be configured on the Master Server to deliver the Daily DBA Summary Reports. We recommend a dedicated O365 or SendGrid SMTP relay.

No OS Agents

OpsDBA requires zero Windows-level agents or third-party executables. No reboots required for installation or maintenance.

Mixed Mode Auth

Required for Master-to-Client communication. We use a secure, low-privilege SQL Login to bridge connections across disparate domains or cloud providers.

Security by Redaction

We sanitize your data before it ever leaves your Master Server.

Edge-Side Redaction

Our logic identifies PII (Personally Identifiable Information) in your error logs locally. Before sending a log to the AI engine for diagnosis, we scrub:

  • Public & Private IP Addresses
  • Domain & Local Usernames
  • Email Addresses & File Paths

Optional Feature

AI Diagnostics Configuration

The AI engine is disabled by default. If enabled, the Master Server requires outbound HTTPS access to your choice of provider.

Google Gemini

Recommended for high-volume logs. Leverages a 1M token window for full log ingestion.

OpenAI GPT-4o

Standard for complex reasoning. Requires a standard OpenAI API key.

Azure OpenAI

For enterprise compliance. Uses your internal private endpoints via Microsoft Copilot logic.

Ollama (Local)

Air-gapped solution. Models run locally on your server. No data leaves the network.
